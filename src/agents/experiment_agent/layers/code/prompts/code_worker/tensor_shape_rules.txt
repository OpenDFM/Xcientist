Tensor Shape Discipline (PyTorch)

Core rules:
- Always state expected tensor shapes using a consistent convention: (B, ...).
- At module boundaries, assert shapes early with clear error messages.
- When reshaping, prefer `.reshape(...)` only if tensor is contiguous; otherwise use `.contiguous().view(...)`.
- When using attention, document: query/key/value shapes and the final output shape.
- For convolutional models, document: input (B, C, H, W) and output channels/strides/padding.
- For MLPs, document: input feature dim and output dim.

Debug checklist:
- Print or log intermediate shapes for the first batch only.
- If a broadcast happens, explicitly explain which dims broadcast and why.
- If using `einsum`, write the equation and the operand shapes.
- If using `permute`, immediately follow with `.contiguous()` when later calling `.view`.

Testing guidance:
- Add a small unit test to validate tensor shapes for a forward pass.
- Use deterministic seed and small batch sizes for fast shape validation.
