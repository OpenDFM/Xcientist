    The Modality Interface: From Linear Projections to Deep Fusion and Generative Unification

The architectural design of the modality interface—the bridge that connects visual encoders with Large Language Models (LLMs)—has evolved into a central battleground for defining the capabilities of Multimodal Large Language Models (MLLMs). While early efforts, such as <Visual Instruction Tuning>, demonstrated the viability of connecting a visual encoder to an LLM using a simple linear projection layer, the field has rapidly shifted toward more sophisticated architectures. These newer designs aim to handle the "modality gap" more effectively, resulting in a spectrum of interfaces ranging from specialized compression modules to deep, interleaved fusion strategies. This evolution is driven by the fundamental challenge that visual data resides in a continuous, high-dimensional space, whereas language operates on discrete, semantic tokens. Bridging this gap requires not just dimensional alignment, but a translation of concepts that preserves spatial relationships and visual semantics. The initial linear projection, while computationally efficient, often acts as a crude bottleneck, forcing the LLM to interpret raw visual features without sufficient context. This limitation spurred the development of intermediate architectures that attempt to inject domain knowledge or learned priors into the mapping process. The tension between efficiency and expressiveness defines the core trade-off in this design space. As models scale, the cost of complex interfaces becomes non-negligible, prompting a re-evaluation of whether heavy pre-processing is always necessary. Consequently, the community is exploring a diverse array of solutions, each optimizing for different constraints such as latency, parameter count, and downstream task performance. This introductory section sets the stage for a detailed analysis of these architectural paradigms, their theoretical underpinnings, and their empirical results.

A prominent approach involves the use of specialized modules to map visual tokens into the semantic space of the LLM. <MiniGPT-4> utilizes Q-Former (inspired by BLIP-2) to learn a set of learnable queries that aggregate visual features before they are fed into the LLM. This method proved that treating visual input as a sequence of compressed, highly relevant features allows an otherwise frozen LLM to understand complex modalities, unlocking emergent capabilities like website generation from sketches. Similarly, <KOSMOS-1> adopts a Perceiver-like architecture to handle interleaved multimodal inputs within a single Transformer. By employing a mapping network and special boundary tokens, it effectively unifies modalities. However, models relying on complex, pre-trained feature compressors like Q-Former (as seen in <BLIVA>, which integrates Q-Former-style embeddings with patch embeddings to better handle text-rich images) often introduce significant parameter overhead and latency.

In reaction to this complexity, recent research has sought to simplify the interface without sacrificing performance. <mPLUG-Owl2> introduces a "modality-adaptive module" that preserves modality-specific features, arguing that strict fusion can degrade text-only capabilities. It decouples the visual pathway while still sharing a universal language decoder. Conversely, <FIBER> argues for "Fusion in the Backbone." Instead of a separate connector, it interleaves cross-attention layers directly into the LLM’s encoder and decoder context. This deep fusion allows the model to reason about visual dependencies dynamically throughout the generation process, rather than relying on a static pre-fusion step. This trade-off—between specialized, compressed encoders (<MiniGPT-4>) and deep, integrated fusion (<FIBER>)—defines the current tension in interface design.

Simultaneously, a radical shift in philosophy has emerged, epitomized by <VisionLLM>. This work proposes treating images as a "foreign language." Rather than designing complex connectors, it aligns vision-centric tasks directly with the LLM’s language processing capabilities. It uses a language-guided image tokenizer and treats outputs (like bounding boxes) as text tokens, effectively unifying the architecture into a pure generative text decoder. This approach suggests that the modality interface might eventually disappear entirely, with the LLM handling all modalities natively. <GIT> further supports this generative unification, achieving state-of-the-art performance in image-to-text generation with a simple encoder-decoder architecture that requires no complex modality-specific modules. This evolution from simple linear projections (<Visual Instruction Tuning>) to specialized compressors (<MiniGPT-4>, <KOSMOS-1>), deep fusion (<FIBER>, <mPLUG-Owl2>), and finally generative unification (<VisionLLM>, <GIT>) represents the field’s search for the most efficient way to bridge the visual and linguistic worlds.

However, architectural innovation cannot compensate for insufficient training data, a point emphasized by <Static Augmentation>. This paper argues that standard supervised fine-tuning (SFT) data is often too narrow, limiting the model's ability to generalize across different instruction styles. This highlights that the "interface" is not just architecture; it is also the instructional data that teaches the model how to utilize that architecture. Even the most elegant connector, such as the "Spatial Vision Aggregator (SVA)" proposed in <Cambrian-1>, relies on a massive, high-quality dataset to fine-tune the interaction between visual backbones and the LLM.

Furthermore, the industry is trending toward architectures that prioritize "visual grounding"—explicitly linking text to visual regions. <KOSMOS-2> introduced grounding capabilities by training on grounded image-text pairs, allowing the model to output <region> tags corresponding to bounding boxes. While not strictly an architectural connector change, it alters the output interface to enforce alignment between the visual and textual streams. Similarly, <UniTAB> Unifies Text and Box outputs, proposing a shared token sequence that allows for grounded captioning. The inclusion of an <obj> token represents a structural change to the interface, enabling the model to explicitly align generated words with visual locations.

The trajectory of research suggests that while early interfaces focused on "how to connect" (via linear layers or projectors), the current focus is on "how to integrate." <mPLUG-Owl2> demonstrates that preserving modality-specific features is crucial for general intelligence, while <FIBER> suggests that dynamic fusion is key for fine-grained tasks. As we look at the evolution from <Visual Instruction Tuning>'s linear layer to <VisionLLM>'s generative decoder, the interface is transforming from a static bridge into a dynamic, intelligent router that determines which modality to prioritize and how to fuse context effectively.

The limitations of current interfaces are also becoming apparent in specific domains. For example, <BLIVA> addresses the failure of generalist connectors in text-rich images by explicitly injecting patch embeddings alongside learned query embeddings. This suggests that "one size fits all" interfaces may not be optimal. As the line between the visual encoder and the language model blurs, exemplified by <Downscaling Intelligence>'s proposal for visual extraction tuning, the modality interface is effectively becoming part of the reasoning engine itself, rather than a mere data conduit.

In conclusion, the modality interface has moved from a trivial linear layer to a core architectural component defining the model's reasoning capabilities. The choice between heavy, pre-trained connectors and deep, integrated fusion reflects a fundamental debate on whether visual processing should be heuristically compressed before reaching the LLM or treated as an integral part of the reasoning process. Works like <VisionLLM> and <GIT> suggest the ultimate interface may be the absence of one—a unified generative space—while current practical architectures like <Cambrian-1> and <FIBER> are pushing the boundaries of sophisticated, multi-stage fusion to maximize performance on complex reasoning tasks.

    The Modality Interface: From Linear Projections to Deep Fusion and Generative Unification

The architectural design of the modality interface—the bridge that connects visual encoders with Large Language Models (LLMs)—has evolved into a central battleground for defining the capabilities of Multimodal Large Language Models (MLLMs). While early efforts, such as <Visual Instruction Tuning>, demonstrated the viability of connecting a visual encoder to an LLM using a simple linear projection layer, the field has rapidly shifted toward more sophisticated architectures. These newer designs aim to handle the "modality gap" more effectively, resulting in a spectrum of interfaces ranging from specialized compression modules to deep, interleaved fusion strategies. This evolution is driven by the fundamental challenge that visual data resides in a continuous, high-dimensional space, whereas language operates on discrete, semantic tokens. Bridging this gap requires not just dimensional alignment, but a translation of concepts that preserves spatial relationships and visual semantics. The initial linear projection, while computationally efficient, often acts as a crude bottleneck, forcing the LLM to interpret raw visual features without sufficient context. This limitation spurred the development of intermediate architectures that attempt to inject domain knowledge or learned priors into the mapping process. The tension between efficiency and expressiveness defines the core trade-off in this design space. As models scale, the cost of complex interfaces becomes non-negligible, prompting a re-evaluation of whether heavy pre-processing is always necessary. Consequently, the community is exploring a diverse array of solutions, each optimizing for different constraints such as latency, parameter count, and downstream task performance. This introductory section sets the stage for a detailed analysis of these architectural paradigms, their theoretical underpinnings, and their empirical results.

To fully appreciate the significance of these architectural shifts, it is essential to understand the nature of the modality gap itself. Visual encoders, typically Convolutional Neural Networks (CNNs) or Vision Transformers (ViTs), map pixel data into dense feature vectors or token sequences. These representations are optimized for tasks like image classification or object detection, capturing low-level textures and high-level shapes but lacking explicit semantic grounding in the way language models understand it. LLMs, conversely, operate on a discrete vocabulary where each token carries a specific, learned meaning and relationships are established through attention mechanisms over a massive corpus of text. A linear projection attempts to map the visual feature space onto the LLM's embedding space, but this is a many-to-one mapping problem where the rich, continuous information in the visual features is compressed into a fixed number of tokens. This compression often discards crucial spatial details and contextual nuances, leaving the LLM to hallucinate or struggle with tasks requiring precise visual understanding, such as counting objects or reading text. The "modality gap" is therefore not just a dimensional mismatch but a semantic and structural chasm.

Early attempts to bridge this gap went beyond simple linear layers but were still relatively naive. For instance, some models experimented with resizing or pooling visual features to fit a specific token count, but these geometric transformations further distorted the visual information. The realization that the mapping itself needed to be learned was a pivotal moment. This led to the concept of "learnable connectors" or "resamplers" that could adaptively compress visual information into a fixed number of tokens, retaining the most salient features for the LLM. This is where the evolution truly began, moving from a static bridge to a dynamic, intelligent filter.

The architectural design of the modality interface—the bridge that connects visual encoders with Large Language Models (LLMs)—has evolved into a central battleground for defining the capabilities of Multimodal Large Language Models (MLLMs). While early efforts, such as <Visual Instruction Tuning>, demonstrated the viability of connecting a visual encoder to an LLM using a simple linear projection layer, the field has rapidly shifted toward more sophisticated architectures. These newer designs aim to handle the "modality gap" more effectively, resulting in a spectrum of interfaces ranging from specialized compression modules to deep, interleaved fusion strategies. This evolution is driven by the fundamental challenge that visual data resides in a continuous, high-dimensional space, whereas language operates on discrete, semantic tokens. Bridging this gap requires not just dimensional alignment, but a translation of concepts that preserves spatial relationships and visual semantics. The initial linear projection, while computationally efficient, often acts as a crude bottleneck, forcing the LLM to interpret raw visual features without sufficient context. This limitation spurred the development of intermediate architectures that attempt to inject domain knowledge or learned priors into the mapping process. The tension between efficiency and expressiveness defines the core trade-off in this design space. As models scale, the cost of complex interfaces becomes non-negligible, prompting a re-evaluation of whether heavy pre-processing is always necessary. Consequently, the community is exploring a diverse array of solutions, each optimizing for different constraints such as latency, parameter count, and downstream task performance. This introductory section sets the stage for a detailed analysis of these architectural paradigms, their theoretical underpinnings, and their empirical results.

To fully appreciate the significance of these architectural shifts, it is essential to understand the nature of the modality gap itself. Visual encoders, typically Convolutional Neural Networks (CNNs) or Vision Transformers (ViTs), map pixel data into dense feature vectors or token sequences. These representations are optimized for tasks like image classification or object detection, capturing low-level textures and high-level shapes but lacking explicit semantic grounding in the way language models understand it. LLMs, conversely, operate on a discrete vocabulary where each token carries a specific, learned meaning and relationships are established through attention mechanisms over a massive corpus of text. A linear projection attempts to map the visual feature space onto the LLM's embedding space, but this is a many-to-one mapping problem where the rich, continuous information in the visual features is compressed into a fixed number of tokens. This compression often discards crucial spatial details and contextual nuances, leaving the LLM to hallucinate or struggle with tasks requiring precise visual understanding, such as counting objects or reading text. The "modality gap" is therefore not just a dimensional mismatch but a semantic and structural chasm.

Early attempts to bridge this gap went beyond simple linear layers but were still relatively naive. For instance, some models experimented with resizing or pooling visual features to fit a specific token count, but these geometric transformations further distorted the visual information. The realization that the mapping itself needed to be learned was a pivotal moment. This led to the concept of "learnable connectors" or "resamplers" that could adaptively compress visual information into a fixed number of tokens, retaining the most salient features for the LLM. This is where the evolution truly began, moving from a static bridge to a dynamic, intelligent filter.

A prominent approach involves the use of specialized modules to map visual tokens into the semantic space of the LLM. <MiniGPT-4> utilizes Q-Former (inspired by BLIP-2) to learn a set of learnable queries that aggregate visual features before they are fed into the LLM. This method proved that treating visual input as a sequence of compressed, highly relevant features allows an otherwise frozen LLM to understand complex modalities, unlocking emergent capabilities like website generation from sketches. Similarly, <KOSMOS-1> adopts a Perceiver-like architecture to handle interleaved multimodal inputs within a single Transformer. By employing a mapping network and special boundary tokens, it effectively unifies modalities. Instead of a separate connector, it interleaves cross-attention layers directly into the LLM’s encoder and decoder context. This deep fusion allows the model to reason about visual dependencies dynamically throughout the generation process, rather than relying on a static pre-fusion step. This trade-off—between specialized, compressed encoders (<MiniGPT-4>) and deep, integrated fusion (<FIBER>)—defines the current tension in interface design.

Simultaneously, a radical shift in philosophy has emerged, epitomized by <VisionLLM>. This work proposes treating images as a "foreign language." Rather than designing complex connectors, it aligns vision-centric tasks directly with the LLM’s language processing capabilities. This approach suggests that the modality interface might eventually disappear entirely, with the LLM handling all modalities natively. <GIT> further supports this generative unification, achieving state-of-the-art performance in image-to-text generation with a simple encoder-decoder architecture that requires no complex modality-specific modules. This evolution from simple linear projections (<Visual Instruction Tuning>) to specialized compressors (<MiniGPT-4>, <KOSMOS-1>), deep fusion (<FIBER>, <mPLUG-Owl2>), and finally generative unification (<VisionLLM>, <GIT>) represents the field’s search for the most efficient way to bridge the visual and linguistic worlds.

The architectural design of the modality interface—the bridge that connects visual encoders with Large Language Models (LLMs)—has evolved into a central battleground for defining the capabilities of Multimodal Large Language Models (MLLMs). While early efforts, such as <Visual Instruction Tuning>, demonstrated the viability of connecting a visual encoder to an LLM using a simple linear projection layer, the field has rapidly shifted toward more sophisticated architectures. These newer designs aim to handle the "modality gap" more effectively, resulting in a spectrum of interfaces ranging from specialized compression modules to deep, interleaved fusion strategies. This evolution is driven by the fundamental challenge that visual data resides in a continuous, high-dimensional space, whereas language operates on discrete, semantic tokens. Bridging this gap requires not just dimensional alignment, but a translation of concepts that preserves spatial relationships and visual semantics. The initial linear projection, while computationally efficient, often acts as a crude bottleneck, forcing the LLM to interpret raw visual features without sufficient context. This limitation spurred the development of intermediate architectures that attempt to inject domain knowledge or learned priors into the mapping process. The tension between efficiency and expressiveness defines the core trade-off in this design space. As models scale, the cost of complex interfaces becomes non-negligible, prompting a re-evaluation of whether heavy pre-processing is always necessary. Consequently, the community is exploring a diverse array of solutions, each optimizing for different constraints such as latency, parameter count, and downstream task performance.     The Modality Interface: From Linear Projections to Deep Fusion and Generative Unification

The architectural design of the modality interface—the bridge that connects visual encoders with Large Language Models (LLMs)—has evolved into a central battleground for defining the capabilities of Multimodal Large Language Models (MLLMs). While early efforts, such as <Visual Instruction Tuning>, demonstrated the viability of connecting a visual encoder to an LLM using a simple linear projection layer, the field has rapidly shifted toward more sophisticated architectures. These newer designs aim to handle the "modality gap" more effectively, resulting in a spectrum of interfaces ranging from specialized compression modules to deep, interleaved fusion strategies. This evolution is driven by the fundamental challenge that visual data resides in a continuous, high-dimensional space, whereas language operates on discrete, semantic tokens. Bridging this gap requires not just dimensional alignment, but a translation of concepts that preserves spatial relationships and visual semantics. The initial linear projection, while computationally efficient, often acts as a crude bottleneck, forcing the LLM to interpret raw visual features without sufficient context. This limitation spurred the development of intermediate architectures that attempt to inject domain knowledge or learned priors into the mapping process. The tension between efficiency and expressiveness defines the core trade-off in this design space. As models scale, the cost of complex interfaces becomes non-negligible, prompting a re-evaluation of whether heavy pre-processing is always necessary. Consequently, the community is exploring a diverse array of solutions, each optimizing for different constraints such as latency, parameter count, and downstream task performance. This introductory section sets the stage for a detailed analysis of these architectural paradigms, their theoretical underpinnings, and their empirical results.

To fully appreciate the significance of these architectural shifts, it is essential to understand the nature of the modality gap itself. Visual encoders, typically Convolutional Neural Networks (CNNs) or Vision Transformers (ViTs), map pixel data into dense feature vectors or token sequences. These representations are optimized for tasks like image classification or object detection, capturing low-level textures and high-level shapes but lacking explicit semantic grounding in the way language models understand it. LLMs, conversely, operate on a discrete vocabulary where each token carries a specific, learned meaning and relationships are established through attention mechanisms over a massive corpus of text. A linear projection attempts to map the visual feature space onto the LLM's embedding space, but this is a many-to-one mapping problem where the rich, continuous information in the visual features is compressed into a fixed number of tokens. This compression often discards crucial spatial details and contextual nuances, leaving the LLM to hallucinate or struggle with tasks requiring precise visual understanding, such as counting objects or reading text. The "modality gap" is therefore not just a dimensional mismatch but a semantic and structural chasm.

Early attempts to bridge this gap went beyond simple linear layers but were still relatively naive. For instance, some models experimented with resizing or pooling visual features to fit a specific token count, but these geometric transformations further distorted the visual information. The realization that the mapping itself needed to be learned was a pivotal moment. This led to the concept of "learnable connectors" or "resamplers" that could adaptively compress visual information into a fixed number of tokens, retaining the most salient features for the LLM. This is where the evolution truly began, moving from a static bridge to a dynamic, intelligent filter.

A prominent approach involves the use of specialized modules to map visual tokens into the semantic space of the LLM. <MiniGPT-4> utilizes Q-Former (inspired by BLIP-2) to learn a set of learnable queries that aggregate visual features before they are fed into the LLM. This method proved that treating visual input as a sequence of compressed, highly relevant features allows an otherwise frozen LLM to understand complex modalities, unlocking emergent capabilities like website generation from sketches. Similarly, <KOSMOS-1> adopts a Perceiver-like architecture to handle interleaved multimodal inputs within a single Transformer. By employing a mapping network and special boundary tokens, it effectively unifies modalities. However, models relying on complex, pre-trained feature compressors like Q-Former (as seen in <BLIVA>, which integrates Q-Former-style embeddings with patch embeddings to better handle text-rich images) often introduce significant parameter overhead and latency.

In reaction to this complexity, recent research has sought to simplify the interface without sacrificing performance. <mPLUG-Owl2> introduces a "modality-adaptive module" that preserves modality-specific features, arguing that strict fusion can degrade text-only capabilities. It decouples the visual pathway while still sharing a universal language decoder. Conversely, <FIBER> argues for "Fusion in the Backbone." Instead of a separate connector, it interleaves cross-attention layers directly into the LLM’s encoder and decoder context. This deep fusion allows the model to reason about visual dependencies dynamically throughout the generation process, rather than relying on a static pre-fusion step. This trade-off—between specialized, compressed encoders (<MiniGPT-4>) and deep, integrated fusion (<FIBER>)—defines the current tension in interface design.

Simultaneously, a radical shift in philosophy has emerged, epitomized by <VisionLLM>. This work proposes treating images as a "foreign language." Rather than designing complex connectors, it aligns vision-centric tasks directly with the LLM’s language processing capabilities. It uses a language-guided image tokenizer and treats outputs (like bounding boxes) as text tokens, effectively unifying the architecture into a pure generative text decoder. This approach suggests that the modality interface might eventually disappear entirely, with the LLM handling all modalities natively. <GIT> further supports this generative unification, achieving state-of-the-art performance in image-to-text generation with a simple encoder-decoder architecture that requires no complex modality-specific modules. This evolution from simple linear projections (<Visual Instruction Tuning>) to specialized compressors (<MiniGPT-4>, <KOSMOS-1>), deep fusion (<FIBER>, <mPLUG-Owl2>), and finally generative unification (<VisionLLM>, <GIT>) represents the field’s search for the most efficient way to bridge the visual and linguistic worlds.

However, architectural innovation cannot compensate for insufficient training data, a point emphasized by <Static Augmentation>. This paper argues that standard supervised fine-tuning (SFT) data is often too narrow, limiting the model's ability to generalize across different instruction styles. This highlights that the "interface" is not just architecture; it is also the instructional data that teaches the model how to utilize that architecture. Even the most elegant connector, such as the "Spatial Vision Aggregator (SVA)" proposed in <Cambrian-1>, relies on a massive, high-quality dataset to fine-tune the interaction between visual backbones and the LLM.

Furthermore, the industry is trending toward architectures that prioritize "visual grounding"—explicitly linking text to visual regions. <KOSMOS-2> introduced grounding capabilities by training on grounded image-text pairs, allowing the model to output <region> tags corresponding to bounding boxes. While not strictly an architectural connector change, it alters the output interface to enforce alignment between the visual and textual streams. Similarly, <UniTAB> Unifies Text and Box outputs, proposing a shared token sequence that allows for grounded captioning. The inclusion of an <obj> token represents a structural change to the interface, enabling the model to explicitly align generated words with visual locations.

The trajectory of research suggests that while early interfaces focused on "how to connect" (via linear layers or projectors), the current focus is on "how to integrate." <mPLUG-Owl2> demonstrates that preserving modality-specific features is crucial for general intelligence, while <FIBER> suggests that dynamic fusion is key for fine-grained tasks. As we look at the evolution from <Visual Instruction Tuning>'s linear layer to <VisionLLM>'s generative decoder, the interface is transforming from a static bridge into a dynamic, intelligent router that determines which modality to prioritize and how to fuse context effectively.

The limitations of current interfaces are also becoming apparent in specific domains. For example, <BLIVA> addresses the failure of generalist connectors in text-rich images by explicitly injecting patch embeddings alongside learned query embeddings. This suggests that "one size fits all" interfaces may not be optimal. As the line between the visual encoder and the language model blurs, exemplified by <Downscaling Intelligence>'s proposal for visual extraction tuning, the modality interface is effectively becoming part of the reasoning engine itself, rather than a mere data conduit.

In conclusion, the modality interface has moved from a trivial linear layer to a core architectural component defining the model's reasoning capabilities. The choice between heavy, pre-trained connectors and deep, integrated fusion reflects a fundamental debate on whether visual processing should be heuristically compressed before reaching the LLM or treated as an integral part of the reasoning process. Works like <VisionLLM> and <GIT> suggest the ultimate interface may be the absence of one—a unified generative space—while current practical architectures like <Cambrian-1> and <FIBER> are pushing the boundaries of sophisticated, multi-stage fusion to maximize performance on complex reasoning tasks.

While early efforts, such as <Visual Instruction Tuning>, demonstrated the viability of connecting a visual encoder to an LLM using a simple linear projection layer, the field has rapidly shifted toward more sophisticated architectures. These newer designs aim to handle the "modality gap" more effectively, resulting in a spectrum of interfaces ranging from specialized compression modules to deep, interleaved fusion strategies. This evolution is driven by the fundamental challenge that visual data resides in a continuous, high-dimensional space, whereas language operates on discrete, semantic tokens. Bridging this gap requires not just dimensional alignment, but a translation of concepts that preserves spatial relationships and visual semantics. The initial linear projection, while computationally efficient, often acts as a crude bottleneck, forcing the LLM to interpret raw visual features without sufficient context. This limitation spurred the development of intermediate architectures that attempt to inject domain knowledge or learned priors into the mapping process. The tension between efficiency and expressiveness defines the core trade-off in this design space. As models scale, the cost of complex interfaces becomes non-negligible, prompting a re-evaluation of whether heavy pre-processing is always necessary. Consequently, the community is exploring a diverse array of solutions, each optimizing for different constraints such as latency, parameter count, and downstream task performance. This introductory section sets the stage for a detailed analysis of these architectural paradigms, their theoretical underpinnings, and their empirical results.

To fully appreciate the significance of these architectural shifts, it is essential to understand the nature of the modality gap itself. Visual encoders, typically Convolutional Neural Networks (CNNs) or Vision Transformers (ViTs), map pixel data into dense feature vectors or token sequences. These representations are optimized for tasks like image classification or object detection, capturing low-level textures and high-level shapes but lacking explicit semantic grounding in the way language models understand it. LLMs, conversely, operate on a discrete vocabulary where each token carries a specific, learned meaning and relationships are established through attention mechanisms over a massive corpus of text. This compression often discards crucial spatial details and contextual nuances, leaving the LLM to hallucinate or struggle with tasks requiring precise visual understanding, such as counting objects or reading text. The "modality gap" is therefore not just a dimensional mismatch but a semantic and structural chasm.

Early attempts to bridge this gap went beyond simple linear layers but were still relatively naive. For instance, some models experimented with resizing or pooling visual features to fit a specific token count, but these geometric transformations further distorted the visual information. The realization that the mapping itself needed to be learned was a pivotal moment. This led to the concept of "learnable connectors" or "resamplers" that could adaptively compress visual information into a fixed number of tokens, retaining the most salient features for the LLM.     The Modality Interface: From Linear Projections to Deep Fusion and Generative Unification

The architectural design of the modality interface—the bridge that connects visual encoders with Large Language Models (LLMs)—has evolved into a central battleground for defining the capabilities of Multimodal Large Language Models (MLLMs). While early efforts, such as <Visual Instruction Tuning>, demonstrated the viability of connecting a visual encoder to an LLM using a simple linear projection layer, the field has rapidly shifted toward more sophisticated architectures. These newer designs aim to handle the "modality gap" more effectively, resulting in a spectrum of interfaces ranging from specialized compression modules to deep, interleaved fusion strategies. This evolution is driven by the fundamental challenge that visual data resides in a continuous, high-dimensional space, whereas language operates on discrete, semantic tokens. Bridging this gap requires not just dimensional alignment, but a translation of concepts that preserves spatial relationships and visual semantics. The initial linear projection, while computationally efficient, often acts as a crude bottleneck, forcing the LLM to interpret raw visual features without sufficient context. This limitation spurred the development of intermediate architectures that attempt to inject domain knowledge or learned priors into the mapping process. The tension between efficiency and expressiveness defines the core trade-off in this design space. As models scale, the cost of complex interfaces becomes non-negligible, prompting a re-evaluation of whether heavy pre-processing is always necessary. Consequently, the community is exploring a diverse array of solutions, each optimizing for different constraints such as latency, parameter count, and downstream task performance. This introductory section sets the stage for a detailed analysis of these architectural paradigms, their theoretical underpinnings, and their empirical results.

The significance of this architectural evolution cannot be overstated, as it directly dictates the upper bound of an MLLM's cognitive abilities. A suboptimal interface acts as an information bottleneck, effectively starving the LLM of the rich visual context necessary for complex reasoning. Conversely, a well-designed interface can unlock emergent capabilities that were not explicitly trained for, such as interpreting complex charts, understanding nuanced spatial relationships, or following visual humor. The research community has therefore moved beyond treating the interface as a mere engineering detail, elevating it to a primary object of study. This shift is characterized by a move from static, deterministic mappings to dynamic, data-driven processes that attempt to understand the intent of the visual query. The following sections will dissect this journey, tracing the lineage from simple linear layers to the cutting-edge paradigms that define the current state of the art.

To fully appreciate the significance of these architectural shifts, it is essential to understand the nature of the modality gap itself. Visual encoders, typically Convolutional Neural Networks (CNNs) or Vision Transformers (ViTs), map pixel data into dense feature vectors or token sequences. These representations are optimized for tasks like image classification or object detection, capturing low-level textures and high-level shapes but lacking explicit semantic grounding in the way language models understand it. LLMs, conversely, operate on a discrete vocabulary where each token carries a specific, learned meaning and relationships are established through attention mechanisms over a massive corpus of text. A linear projection attempts to map the visual feature space onto the LLM's embedding space, but this is a many-to-one mapping problem where the rich, continuous information in the visual features is compressed into a fixed number of tokens. This compression often discards crucial spatial details and contextual nuances, leaving the LLM to hallucinate or struggle with tasks requiring precise visual understanding, such as counting objects or reading text. The "modality gap" is therefore not just a dimensional mismatch but a semantic and structural chasm.

Early attempts to bridge this gap went beyond simple linear layers but were still relatively naive. For instance, some models experimented with resizing or pooling visual features to fit a specific token count, but these geometric transformations further distorted the visual information. The realization that the mapping itself needed to be learned was a pivotal moment. This led to the concept of "learnable connectors" or "resamplers" that could adaptively compress visual information into a fixed number of tokens, retaining the most salient features for the LLM. This is where the evolution truly began, moving from a static bridge to a dynamic, intelligent filter.

A prominent approach involves the use of specialized modules to map visual tokens into the semantic space of the LLM. <MiniGPT-4> utilizes Q-Former (inspired by BLIP-2) to learn a set of learnable queries that aggregate visual features before they are fed into the LLM. This method proved that treating visual input as a sequence of compressed, highly relevant features allows an otherwise frozen LLM to understand complex modalities, unlocking emergent capabilities like website generation from sketches. Similarly, <KOSMOS-1> adopts a Perceiver-like architecture to handle interleaved multimodal inputs within a single Transformer. By employing a mapping network and special boundary tokens, it effectively unifies modalities. However, models relying on complex, pre-trained feature compressors like Q-Former (as seen in <BLIVA>, which integrates Q-Former-style embeddings with patch embeddings to better handle text-rich images) often introduce significant parameter overhead and latency.

In reaction to this complexity, recent research has sought to simplify the interface without sacrificing performance. <mPLUG-Owl2> introduces a "modality-adaptive module" that preserves modality-specific features, arguing that strict fusion can degrade text-only capabilities. It decouples the visual pathway while still sharing a universal language decoder. Conversely, <FIBER> argues for "Fusion in the Backbone." Instead of a separate connector, it interleaves cross-attention layers directly into the LLM’s encoder and decoder context. This deep fusion allows the model to reason about visual dependencies dynamically throughout the generation process, rather than relying on a static pre-fusion step. This trade-off—between specialized, compressed encoders (<MiniGPT-4>) and deep, integrated fusion (<FIBER>)—defines the current tension in interface design.

Simultaneously, a radical shift in philosophy has emerged, epitomized by <VisionLLM>. This work proposes treating images as a "foreign language." Rather than designing complex connectors, it aligns vision-centric tasks directly with the LLM’s language processing capabilities. It uses a language-guided image tokenizer and treats outputs (like bounding boxes) as text tokens, effectively unifying the architecture into a pure generative text decoder. This approach suggests that the modality interface might eventually disappear entirely, with the LLM handling all modalities natively. <GIT> further supports this generative unification, achieving state-of-the-art performance in image-to-text generation with a simple encoder-decoder architecture that requires no complex modality-specific modules. This evolution from simple linear projections (<Visual Instruction Tuning>) to specialized compressors (<MiniGPT-4>, <KOSMOS-1>), deep fusion (<FIBER>, <mPLUG-Owl2>), and finally generative unification (<VisionLLM>, <GIT>) represents the field’s search for the most efficient way to bridge the visual and linguistic worlds.

However, architectural innovation cannot compensate for insufficient training data, a point emphasized by <Static Augmentation>. This paper argues that standard supervised fine-tuning (SFT) data is often too narrow, limiting the model's ability to generalize across different instruction styles. This highlights that the "interface" is not just architecture; it is also the instructional data that teaches the model how to utilize that architecture. Even the most elegant connector, such as the "Spatial Vision Aggregator (SVA)" proposed in <Cambrian-1>, relies on a massive, high-quality dataset to fine-tune the interaction between visual backbones and the LLM.

Furthermore, the industry is trending toward architectures that prioritize "visual grounding"—explicitly linking text to visual regions. <KOSMOS-2> introduced grounding capabilities by training on grounded image-text pairs, allowing the model to output <region> tags corresponding to bounding boxes. While not strictly an architectural connector change, it alters the output interface to enforce alignment between the visual and textual streams. Similarly, <UniTAB> Unifies Text and Box outputs, proposing a shared token sequence that allows for grounded captioning. The inclusion of an <obj> token represents a structural change to the interface, enabling the model to explicitly align generated words with visual locations.

The trajectory of research suggests that while early interfaces focused on "how to connect" (via linear layers or projectors), the current focus is on "how to integrate." <mPLUG-Owl2> demonstrates that preserving modality-specific features is crucial for general intelligence, while <FIBER> suggests that dynamic fusion is key for fine-grained tasks. As we look at the evolution from <Visual Instruction Tuning>'s linear layer to <VisionLLM>'s generative decoder, the interface is transforming from a static bridge into a dynamic, intelligent router that determines which modality to prioritize and how to fuse context effectively.

The limitations of current interfaces are also becoming apparent in specific domains. For example, <BLIVA> addresses the failure of generalist connectors in text-rich images by explicitly injecting patch embeddings alongside learned query embeddings. This suggests that "one size fits all" interfaces may not be optimal. As the line between the visual encoder and the language model blurs, exemplified by <Downscaling Intelligence>'s proposal for visual extraction tuning, the modality interface is effectively becoming part of the reasoning engine itself, rather than a mere data conduit.

In conclusion, the modality interface has moved from a trivial linear layer to a core architectural component defining the model's reasoning capabilities. The choice between heavy, pre-trained connectors and deep, integrated fusion reflects a fundamental debate on whether visual processing should be heuristically compressed before reaching the LLM or treated as an integral part of the reasoning process. Works like <VisionLLM> and <GIT> suggest the ultimate interface may be the absence of one—a unified generative space—while current practical architectures like <Cambrian-1> and <FIBER> are pushing the boundaries of sophisticated, multi-stage fusion to maximize performance on complex reasoning tasks.

LLMs, conversely, operate on a discrete vocabulary where each token carries a specific, learned meaning and relationships are established through attention mechanisms over a massive corpus of text. A linear projection attempts to map the visual feature space onto the LLM's embedding space, but this is a many-to-one mapping problem where the rich, continuous information in the visual features is compressed into a fixed number of tokens. This compression often discards crucial spatial details and contextual nuances, leaving the LLM to hallucinate or struggle with tasks requiring precise visual understanding, such as counting objects or reading text. The "modality gap" is therefore not just a dimensional mismatch but a semantic and structural chasm.

Early attempts to bridge this gap went beyond simple linear layers but were still relatively naive. For instance, some models experimented with resizing or pooling visual features to fit a specific token count, but these geometric transformations further distorted the visual information. The realization that the mapping itself needed to be learned was a pivotal moment. This led to the concept of "learnable connectors" or "resamplers" that could adaptively compress visual information into a fixed number of tokens, retaining the most salient features for the LLM. This is where the evolution truly began, moving from a static bridge to a dynamic, intelligent filter.

A prominent approach involves the use of specialized modules to map visual tokens into the semantic space of the LLM. <MiniGPT-4> utilizes Q-Former (inspired by BLIP-2) to learn a set of learnable queries that aggregate visual features before they are fed into the LLM. This method proved that treating visual input as a sequence of compressed, highly relevant features allows an otherwise frozen LLM to understand complex modalities, unlocking emergent capabilities like website generation from sketches. Similarly, <KOSMOS-1> adopts a Perceiver-like architecture to handle interleaved multimodal inputs within a single Transformer. By employing a mapping network and special boundary tokens, it effectively unifies modalities. However, models relying on complex, pre-trained feature compressors like Q-Former (as seen in <BLIVA>, which integrates Q-Former-style embeddings with patch embeddings to better handle text-rich images) often introduce significant parameter overhead and latency.

In reaction to this complexity, recent research has sought to simplify the interface without sacrificing performance. <mPLUG-Owl2> introduces a "modality-adaptive module" that preserves modality-specific features, arguing that strict fusion can degrade text-only capabilities. It decouples the visual pathway while still sharing a universal language decoder. Conversely, <FIBER> argues for "Fusion in the Backbone." Instead of a separate connector, it interleaves cross-attention layers directly into the LLM’s encoder and decoder context. This deep fusion allows the model to reason about visual dependencies dynamically throughout the generation process, rather than relying on a static pre-fusion step. This trade-off—between specialized, compressed encoders (<MiniGPT-4>) and deep, integrated fusion (<FIBER>)—defines the current tension in interface design.

Rather than designing complex connectors, it aligns vision-centric tasks directly with the LLM’s language processing capabilities. This approach suggests that the modality interface might eventually disappear entirely, with the LLM handling all modalities natively. <GIT> further supports this generative unification, achieving state-of-the-art performance in image-to-text generation with a simple encoder-decoder architecture that requires no complex modality-specific modules. This evolution from simple linear projections (<Visual Instruction Tuning>) to specialized compressors (<MiniGPT-4>, <KOSMOS-1>), deep fusion (<FIBER>, <mPLUG-Owl2>), and finally generative unification (<VisionLLM>, <GIT>) represents the field’s search for the most efficient way to bridge the visual and linguistic worlds.

This paper argues that standard supervised fine-tuning (SFT) data is often too narrow, limiting the model's ability to generalize across different instruction styles. This highlights that the "interface" is not just architecture; it is also the instructional data that teaches the model how to utilize that architecture. Even the most elegant connector, such as the "Spatial Vision Aggregator (SVA)" proposed in <Cambrian-1>, relies on a massive, high-quality dataset to fine-tune the interaction between visual backbones and the LLM.

Furthermore, the industry is trending toward architectures that prioritize "visual grounding"—explicitly linking text to visual regions. <KOSMOS-2> introduced grounding capabilities by training on grounded image-text pairs, allowing the model to output <region> tags corresponding to bounding boxes. Similarly, <UniTAB> Unifies Text and Box outputs, proposing a shared token sequence that allows for grounded captioning. The inclusion of an <obj> token represents a structural change to the interface, enabling the model to explicitly align generated words with visual locations.

The trajectory of research suggests that while early interfaces focused on "how to connect" (via linear layers or projectors), the current focus is on "how to integrate." <mPLUG-Owl2> demonstrates that preserving modality-specific features is crucial for general intelligence, while <FIBER> suggests that dynamic fusion is key for fine-grained tasks. This suggests that "one size fits all" interfaces may not be optimal.     The Modality Interface: From Linear Projections to Deep Fusion and Generative Unification

The architectural design of the modality interface—the bridge that connects visual encoders with Large Language Models (LLMs)—has evolved into a central battleground for defining the capabilities of Multimodal Large Language Models (MLLMs). While early efforts, such as <Visual Instruction Tuning>, demonstrated the viability of connecting a visual encoder to an LLM using a simple linear projection layer, the field has rapidly shifted toward more sophisticated architectures. These newer designs aim to handle the "modality gap" more effectively, resulting in a spectrum of interfaces ranging from specialized compression modules to deep, interleaved fusion strategies. This evolution is driven by the fundamental challenge that visual data resides in a continuous, high-dimensional space, whereas language operates on discrete, semantic tokens. Bridging this gap requires not just dimensional alignment, but a translation of concepts that preserves spatial relationships and visual semantics. The initial linear projection, while computationally efficient, often acts as a crude bottleneck, forcing the LLM to interpret raw visual features without sufficient context. This limitation spurred the development of intermediate architectures that attempt to inject domain knowledge or learned priors into the mapping process. The tension between efficiency and expressiveness defines the core trade-off in this design space. As models scale, the cost of complex interfaces becomes non-negligible, prompting a re-evaluation of whether heavy pre-processing is always necessary. Consequently, the community is exploring a diverse array of solutions, each optimizing for different constraints such as latency, parameter count, and downstream task performance. This introductory section sets the stage for a detailed analysis of these architectural paradigms, their theoretical underpinnings, and their empirical results.

To fully appreciate the significance of these architectural shifts, it is essential to understand the nature of the modality gap itself. Visual encoders, typically Convolutional Neural Networks (CNNs) or Vision Transformers (ViTs), map pixel data into dense feature vectors or token sequences. These representations are optimized for tasks like image classification or object detection, capturing low-level textures and high-level shapes but lacking explicit semantic grounding in the way language models understand it. LLMs, conversely, operate on a discrete vocabulary where each token carries a specific, learned meaning and relationships are established through attention mechanisms over a massive corpus of text. A linear projection attempts to map the visual feature space onto the LLM's embedding space, but this is a many-to-one mapping problem where the rich, continuous information in the visual features is compressed into a fixed number of tokens. This compression often discards crucial spatial details and contextual nuances, leaving the LLM to hallucinate or struggle with tasks requiring precise visual understanding, such as counting objects or reading text. The "modality gap" is therefore not just a dimensional mismatch but a semantic and structural chasm.

Early attempts to bridge this gap went beyond simple linear layers but were still relatively naive. For instance, some models experimented with resizing or pooling visual features to fit a specific token count, but these geometric transformations further distorted the visual information. The realization that the mapping itself needed to be learned was a pivotal moment. This led to the concept of "learnable connectors" or "resamplers" that could adaptively compress visual information into a fixed number of tokens, retaining the most salient features for the LLM. This is where the evolution truly began, moving from a static bridge to a dynamic, intelligent filter.

A prominent approach involves the use of specialized modules to map visual tokens into the semantic space of the LLM. <MiniGPT-4> utilizes Q-Former (inspired by BLIP-2) to learn a set of learnable queries that aggregate visual features before they are fed into the LLM. This method proved that treating visual input as a sequence of compressed, highly relevant features allows an otherwise frozen LLM to understand complex modalities, unlocking emergent capabilities like website generation from sketches. Similarly, <KOSMOS-1> adopts a Perceiver-like architecture to handle interleaved multimodal inputs within a single Transformer. By employing a mapping network and special boundary tokens, it effectively unifies modalities. However, models relying on complex, pre-trained feature compressors like Q-Former (as seen in <BLIVA>, which integrates Q-Former-style embeddings with patch embeddings to better handle text-rich images) often introduce significant parameter overhead and latency.

In reaction to this complexity, recent research has sought to simplify the interface without sacrificing performance. <mPLUG-Owl2> introduces a "modality-adaptive module" that preserves modality-specific features, arguing that strict fusion can degrade text-only capabilities. It decouples the visual pathway while still sharing a universal language decoder. Conversely, <FIBER> argues for "Fusion in the Backbone." Instead of a separate connector, it interleaves cross-attention layers directly into the LLM’s encoder and decoder context. This deep fusion allows the model to reason about visual dependencies dynamically throughout the generation process, rather than relying on a static pre-fusion step. This trade-off—between specialized, compressed encoders (<MiniGPT-4>) and deep, integrated fusion (<FIBER>)—defines the current tension in interface design.

Simultaneously, a radical shift in philosophy has emerged, epitomized by <VisionLLM>. This work proposes treating images as a "foreign language." Rather than designing complex connectors, it aligns vision-centric tasks directly with the LLM’s language processing capabilities. It uses a language-guided image tokenizer and treats outputs (like bounding boxes) as text tokens, effectively unifying the architecture into a pure generative text decoder. This approach suggests that the modality interface might eventually disappear entirely, with the LLM handling all modalities natively. <GIT> further supports this generative unification, achieving state-of-the-art performance in image-to-text generation with a simple encoder-decoder architecture that requires no complex modality-specific modules. This evolution from simple linear projections (<Visual Instruction Tuning>) to specialized compressors (<MiniGPT-4>, <KOSMOS-1>), deep fusion (<FIBER>, <mPLUG-Owl2>), and finally generative unification (<VisionLLM>, <GIT>) represents the field’s search for the most efficient way to bridge the visual and linguistic worlds.

However, architectural innovation cannot compensate for insufficient training data, a point emphasized by <Static Augmentation>. This paper argues that standard supervised fine-tuning (SFT) data is often too narrow, limiting the model's ability to generalize across different instruction styles. This highlights that the "interface" is not just architecture; it is also the instructional data that teaches the model how to utilize that architecture. Even the most elegant connector, such as the "Spatial Vision Aggregator (SVA)" proposed in <Cambrian-1>, relies on a massive, high-quality dataset to fine-tune the interaction between visual backbones and the LLM.

Furthermore, the industry is trending toward architectures that prioritize "visual grounding"—explicitly linking text to visual regions. <KOSMOS-2> introduced grounding capabilities by training on grounded image-text pairs, allowing the model to output <region> tags corresponding to bounding boxes. While not strictly an architectural connector change, it alters the output interface to enforce alignment between the visual and textual streams. Similarly, <UniTAB> Unifies Text and Box outputs, proposing a shared token sequence that allows for grounded captioning. The inclusion of an <obj> token represents a structural change to the interface, enabling the model to explicitly align generated words with visual locations.

The trajectory of research suggests that while early interfaces focused on "how to connect" (via linear layers or projectors), the current focus is on "how to integrate." <mPLUG-Owl2> demonstrates that preserving modality-specific features is crucial for general intelligence, while <FIBER> suggests that dynamic fusion is key for fine-grained tasks. As we look at the evolution from <Visual Instruction Tuning>'s linear layer to <VisionLLM>'s generative decoder, the interface is transforming from a static bridge into a dynamic, intelligent router that determines which modality to prioritize and how to fuse context effectively.

The limitations of current interfaces are also becoming apparent in specific domains. For example, <BLIVA> addresses the failure of generalist connectors in text-rich images by explicitly injecting patch embeddings alongside learned query embeddings. This suggests that "one size fits all" interfaces may not be optimal. As the line between the visual encoder and the language model blurs, exemplified by <Downscaling Intelligence>'s proposal for visual extraction tuning, the modality interface is effectively becoming part of the reasoning engine itself, rather than a mere data conduit.

In conclusion, the modality interface has moved from a trivial linear layer to a core architectural component defining the model's reasoning capabilities. The choice between heavy, pre-trained connectors and deep, integrated fusion reflects a fundamental debate on whether visual processing should be heuristically compressed before reaching the LLM or treated as an integral part of the reasoning process. Works like <VisionLLM> and <GIT> suggest the ultimate interface may be the absence of one—a unified generative space—while current practical architectures like <Cambrian-1> and <FIBER> are pushing the boundaries of sophisticated, multi-stage fusion to maximize performance on complex reasoning tasks.

The significance of this architectural evolution cannot be overstated, as it directly dictates the upper bound of an MLLM's cognitive abilities. A suboptimal interface acts as an information bottleneck, effectively starving the LLM of the rich visual context necessary for complex reasoning. Conversely, a well-designed interface can unlock emergent capabilities that were not explicitly trained for, such as interpreting complex charts, understanding nuanced spatial relationships, or following visual humor. The research community has therefore moved beyond treating the interface as a mere engineering detail, elevating it to a primary object of study. This shift is characterized by a move from static, deterministic mappings to dynamic, data-driven processes that attempt to understand the intent of the visual query. To fully appreciate the significance of these architectural shifts, it is essential to understand the nature of the modality gap itself. Visual encoders, typically Convolutional Neural Networks (CNNs) or Vision Transformers (ViTs), map pixel data into dense feature vectors or token sequences. These representations are optimized for tasks like image classification or object detection, capturing low-level textures and high-level shapes but lacking explicit semantic grounding in the way language models understand it. LLMs, conversely, operate on a discrete vocabulary where each token carries a specific, learned meaning and relationships are established through attention mechanisms over a massive corpus of text. A linear projection attempts to map the visual feature space onto the LLM's embedding space, but this is a many-to-one mapping problem where the rich, continuous information in the visual features is compressed into a fixed number of tokens. This compression often discards crucial spatial details and contextual nuances, leaving the LLM to hallucinate or struggle with tasks requiring precise visual understanding, such as counting objects or reading text. The "modality gap" is therefore not just a dimensional mismatch but a semantic and structural chasm.

Early attempts to bridge this gap went beyond simple linear layers but were still relatively naive. For instance, some models experimented with resizing or pooling visual features to fit a specific token count, but these geometric transformations further distorted the visual information. The realization that the mapping itself needed to be learned was a pivotal moment. This led to the concept of "learnable connectors" or "resamplers" that could adaptively compress visual information into a fixed number of tokens, retaining the most salient features for the LLM. This is where the evolution truly began, moving from a static bridge to a dynamic, intelligent filter.

A prominent approach involves the use of specialized modules to map visual tokens into the semantic space of the LLM. <MiniGPT-4> utilizes Q-Former (inspired by BLIP-2) to learn a set of learnable queries that aggregate visual features before they are fed into the LLM. This method proved that treating visual input as a sequence of compressed, highly relevant features allows an otherwise frozen LLM to understand complex modalities, unlocking emergent capabilities like website generation from sketches. Similarly, <KOSMOS-1> adopts a Perceiver-like architecture to handle interleaved multimodal inputs within a single Transformer. By employing a mapping network and special boundary tokens, it effectively unifies modalities. However, models relying on complex, pre-trained feature compressors like Q-Former (as seen in <BLIVA>, which integrates Q-Former-style embeddings with patch embeddings to better handle text-rich images) often introduce significant parameter overhead and latency.

In reaction to this complexity, recent research has sought to simplify the interface without sacrificing performance. <mPLUG-Owl2> introduces a "modality-adaptive module" that preserves modality-specific features, arguing that strict fusion can degrade text-only capabilities. It decouples the visual pathway while still sharing a universal language decoder. Conversely, <FIBER> argues for "Fusion in the Backbone." Instead of a separate connector, it interleaves cross-attention layers directly into the LLM’s encoder and decoder context. This deep fusion allows the model to reason about visual dependencies dynamically throughout the generation process, rather than relying on a static pre-fusion step. This trade-off—between specialized, compressed encoders (<MiniGPT-4>) and deep, integrated fusion (<FIBER>)—defines the current tension in interface design.

Simultaneously, a radical shift in philosophy has emerged, epitomized by <VisionLLM>. This work proposes treating images as a "foreign language." Rather than designing complex connectors, it aligns vision-centric tasks directly with the LLM’s language processing capabilities. It uses a language-guided image tokenizer and treats outputs (like bounding boxes) as text tokens, effectively unifying the architecture into a pure generative text decoder. This approach suggests that the modality interface might eventually disappear entirely, with the LLM handling all modalities natively. <GIT> further supports this generative unification, achieving state-of-the-art performance in image-to-text generation with a simple encoder-decoder architecture that requires no complex modality-specific modules. This evolution from simple linear projections (<Visual Instruction Tuning>) to specialized compressors (<MiniGPT-4>, <KOSMOS-1>), deep fusion (<FIBER>, <mPLUG-Owl2>), and finally generative unification (<VisionLLM>, <GIT>) represents the field’s search for the most efficient way to bridge the visual and linguistic worlds.

However, architectural innovation cannot compensate for insufficient training data, a point emphasized by <Static Augmentation>. This paper argues that standard supervised fine-tuning (SFT) data is often too narrow, limiting the model's ability to generalize across different instruction styles. This highlights that the "interface" is not just architecture; it is also the instructional data that teaches the model how to utilize that architecture. Even the most elegant connector, such as the "Spatial Vision Aggregator (SVA)" proposed in <Cambrian-1>, relies on a massive, high-quality dataset to fine-tune the interaction between visual backbones and the LLM.

Furthermore, the industry is trending toward architectures that prioritize "visual grounding"—explicitly linking text to visual regions. <KOSMOS-2> introduced grounding capabilities by training on grounded image-text pairs, allowing the model to output <region> tags corresponding to bounding boxes. While not strictly an architectural connector change, it alters the output interface to enforce alignment between the visual and textual streams. Similarly, <UniTAB> Unifies Text and Box outputs, proposing a shared token sequence that allows for grounded captioning. The inclusion of an <obj> token represents a structural change to the interface, enabling the model to explicitly align generated words with visual locations.

The trajectory of research suggests that while early interfaces focused on "how to connect" (via linear layers or projectors), the current focus is on "how to integrate." <mPLUG-Owl2> demonstrates that preserving modality-specific features is crucial for general intelligence, while <FIBER> suggests that dynamic fusion is key for fine-grained tasks. As we look at the evolution from <Visual Instruction Tuning>'s linear layer to <VisionLLM>'s generative decoder, the interface is transforming from a static bridge into a dynamic, intelligent router that determines which modality to prioritize and how to fuse context effectively.

The limitations of current interfaces are also becoming apparent in specific domains. For example, <BLIVA> addresses the failure of generalist connectors in text-rich images by explicitly injecting patch embeddings alongside learned query embeddings. This suggests that "one size fits all" interfaces may not be optimal. As the line between the visual encoder and the language model blurs, exemplified by <Downscaling Intelligence>'s proposal for visual extraction tuning, the modality interface is effectively becoming part of the reasoning engine itself, rather than a mere data conduit.
